# 变更传播检查清单

> 本清单汇总了通过三次跨阶段修改场景模拟发现的11个流程卡点及其解决方案，供后续写作时对照使用。

---

## 一、变更触发检查

### 1.1 变更分级判定

**判定标准**：

| 级别 | 定义 | 典型场景 | 处理方式 |
|------|------|---------|---------|
| **S级** | 核心设定变更 | 核心冲突改写、主题方向变更、能力体系根本性变更 | 全链路更新 |
| **A级** | 重要设定调整 | 支线情节调整、人物关系调整、社会规则调整 | 更新直接关联文档 |
| **B级** | 细节修改 | 章节标题修改、外貌描述修改、对话措辞修改 | 只更新当前文档 |

**检查项**：
- [ ] 当前变更属于哪个级别？
- [ ] S级变更是否必要？是否可以通过A级或B级变更替代？
- [ ] S级变更是否已执行变更可行性评估？

---

### 1.2 变更可行性评估（S级变更必做）

**评估项**：

| # | 评估问题 | 判断标准 |
|---|---------|---------|
| 1 | 变更是否与核心主题冲突？ | 如果冲突，是否可以调和？ |
| 2 | 变更是否需要重写超过50%的文档？ | 如果是，考虑是否值得 |
| 3 | 变更是否改变故事的根本性质？ | 如果是，需要重新评估 |
| 4 | 变更的修复成本是否远大于收益？ | 如果是，考虑替代方案 |

**评估结果**：
- ✅ 可行：按流程执行
- ⚠️ 有条件可行：修改变更方案后执行
- ❌ 不可行：否决变更，寻找替代方案

---

## 二、影响范围识别检查

### 2.1 四层识别清单

**按层级排查**：

```
第一层：直接关联
├── 世界观：能力体系 → 社会规则 → 历史设定 → 地理环境
├── 人物：人物小传 → 人物关系 → 成长弧线 → 语言风格
├── 大纲：章节大纲 → 冲突地图 → 关键场景 → 伏笔设计
└── 写作：自检清单 → 对话示例

第二层：跨层关联
├── 世界观变更 → 检查人物设定是否受影响
├── 人物变更 → 检查大纲是否受影响
├── 大纲变更 → 检查世界观是否受影响（逆向传播）
└── 任意变更 → 检查写作指南是否受影响

第三层：隐性关联
├── 能力体系变更 → 检查所有人物的能力使用场景
├── 社会规则变更 → 检查所有人物的社会身份
├── 人物关系变更 → 检查所有相关人物的行为逻辑
└── 历史设定变更 → 检查所有涉及历史的情节

第四层：过审关联
├── 任意变更 → 检查是否引入过审风险
├── 能力设定变更 → 检查是否使用迷信敏感词汇
└── 朝代/历史变更 → 检查是否使用真实历史名称
```

**检查项**：
- [ ] 是否已按四层清单逐一排查？
- [ ] 是否遗漏了隐性关联？（如：苏婉清的医者身份与林墨白的身体异变）
- [ ] 是否检查了过审关联？

---

### 2.2 受影响文档列表

**文档分类**：

| 类别 | 典型文档 | 检查要点 |
|------|---------|---------|
| 世界观 | special-systems.md, social-rules.md, history.md | 底层设定是否支撑变更 |
| 人物 | protagonist-*.md, antagonist-*.md, character-relationships.md | 人物行为是否与新设定一致 |
| 大纲 | chapter-outline.md, conflict-map.md, key-scenes.md | 情节是否与新设定一致 |
| 写作 | chapter-self-check-list.md, dialogue-example-*.md | 自检项是否需要补充 |

**检查项**：
- [ ] 是否列出了所有受影响的文档？
- [ ] 是否标注了每个文档的修改优先级？
- [ ] 是否预估了修改工作量？

## 三、关系链断裂检测检查

### 3.1 断裂检测清单（S/A级变更必做）

**检测项**：

| # | 检测问题 | 检查方法 |
|---|---------|---------|
| 1 | 新设定是否与现有角色身份冲突？ | 每个角色的身份/职业/知识是否与新设定矛盾？ |
| 2 | 新设定是否与现有角色关系冲突？ | 角色之间的信息差、互动是否与新设定矛盾？ |
| 3 | 新设定是否与现有情节冲突？ | 已发生的情节、角色的行为是否与新设定矛盾？ |

**典型断裂场景**：

| 场景 | 断裂表现 | 检查方法 |
|------|---------|---------|
| 新增能力设定 | 某角色应该能使用该能力但未使用 | 检查所有角色的能力使用场景 |
| 新增组织设定 | 某角色应该知道该组织但不知道 | 检查所有角色的知识范围 |
| 新增关系设定 | 某角色的行为与新关系矛盾 | 检查所有相关角色的行为逻辑 |
| 修改底层设定 | 上层设定依赖底层设定，底层变了上层未变 | 检查所有依赖该设定的上层设定 |

**检查项**：
- [ ] 是否执行了三项断裂检测？
- [ ] 是否发现了断裂点？
- [ ] 如果发现断裂点，是否分析了断裂点之间的依赖关系？

---

### 3.2 断裂点依赖分析（多个断裂点时）

**分析步骤**：

```
1. 列出所有断裂点
   断裂点A：xxx
   断裂点B：xxx
   断裂点C：xxx

2. 分析依赖关系
   断裂点A → 影响断裂点B
   断裂点B → 影响断裂点C

3. 确定修复顺序
   先修复A → 再修复B → 最后修复C

4. 上游修复方案确定后，再评估下游断裂点
```

**检查项**：
- [ ] 是否画出了断裂点依赖关系图？
- [ ] 是否按依赖顺序排列了修复顺序？
- [ ] 上游修复方案是否已确定？

---

## 四、修复方案评估检查

### 4.1 修复方案评估标准

**评估项**：

| # | 评估维度 | 评估问题 | 权重 |
|---|---------|---------|------|
| 1 | 逻辑自洽性 | 修复方案是否与现有设定逻辑自洽？ | ★★★★★ |
| 2 | 戏剧张力 | 修复方案是否增加了戏剧张力？ | ★★★★☆ |
| 3 | 角色深度 | 修复方案是否深化了角色？ | ★★★★☆ |
| 4 | 过审风险 | 修复方案是否增加过审风险？ | ★★★★★ |
| 5 | 修改成本 | 修复方案需要修改多少文档？ | ★★★☆☆ |

**评分标准**：
- ★★★★★：优秀（强烈推荐）
- ★★★★☆：良好（推荐）
- ★★★☆☆：一般（可接受）
- ★★☆☆☆：较差（需改进）
- ★☆☆☆☆：很差（不推荐）

**检查项**：
- [ ] 是否提出了多个修复方案？
- [ ] 是否对每个方案进行了五维评估？
- [ ] 是否选择了最优方案？

## 五、更新顺序检查

### 5.1 更新顺序策略选择

**策略判定**：

| 传播方向 | 判定条件 | 更新策略 |
|---------|---------|---------|
| **顺向传播** | 变更源头在上游（世界观/人物） | 先更新自身 → 再更新下游 |
| **逆向传播** | 变更源头在下游（大纲/初稿/二稿） | 先评估底层支撑 → 再更新 |

**顺向传播更新顺序**：
```
1. 先更新当前阶段自身
2. 再更新受影响的人物设定
3. 再更新受影响的大纲
4. 最后更新写作指南
```

**逆向传播更新顺序**：
```
1. 先评估底层能否支撑变更
   - 底层可以支撑：按顺向传播顺序执行
   - 底层无法支撑：先修复底层，再更新当前阶段
2. 修复底层后，重新执行底层阶段的门控检查
3. 确认底层修复不会导致其他不一致
4. 再按顺向传播顺序更新
```

**检查项**：
- [ ] 当前变更属于顺向传播还是逆向传播？
- [ ] 如果是逆向传播，是否先评估了底层支撑？
- [ ] 更新顺序是否合理？

---

### 5.2 回溯修改规则（逆向传播必做）

**回溯步骤**：

```
1. 标记回溯原因和范围
   - 回溯原因：xxx
   - 回溯范围：xxx文档

2. 修改完成后重新执行该阶段的门控检查
   - 门控检查项：完整性、一致性、过审检查等

3. 确认修改不会导致该阶段其他文档不一致
   - 检查该阶段所有文档是否仍然一致

4. 只修改受影响的文档，不做"顺便"修改
   - 避免引入新的不一致

5. 修改后必须更新变更记录
   - 记录修改内容、原因、影响范围

6. 从回溯阶段开始，逐阶段检查是否有新的不一致
   - 回溯阶段 → 下一阶段 → 再下一阶段 → ...
```

**检查项**：
- [ ] 是否标记了回溯原因和范围？
- [ ] 修改后是否重新执行了门控检查？
- [ ] 是否确认了不会导致其他不一致？
- [ ] 是否只修改了受影响的文档？
- [ ] 是否更新了变更记录？
- [ ] 是否逐阶段检查了新的不一致？

### 6.2 变更回滚机制

**回滚场景**：

| 场景 | 回滚时机 | 回滚方式 |
|------|---------|---------|
| 变更方向不对 | 变更过程中发现 | 回滚到变更前版本 |
| 验证不通过 | 变更完成后验证失败 | 回滚到变更前版本 |
| 发现更好的方案 | 变更过程中 | 回滚后按新方案重新变更 |

**回滚检查项**：
- [ ] 变更前是否备份了当前版本？
- [ ] 回滚时是否能准确恢复到变更前版本？
- [ ] 回滚后是否确认了没有残留修改？

## 七、跨阶段一致性验证检查

### 7.1 六维度验证清单

**验证项**：

| # | 验证维度 | 验证内容 | 验证方法 |
|---|---------|---------|---------|
| 1 | 世界观内部一致性 | 世界观各设定之间是否逻辑自洽？ | 检查能力体系、社会规则、历史设定是否一致 |
| 2 | 人物设定一致性 | 人物行为是否全部符合设定？ | 检查人物小传、关系、成长弧线是否一致 |
| 3 | 大纲一致性 | 情节是否全部符合设定？ | 检查章节大纲、冲突地图、关键场景是否一致 |
| 4 | 语言风格一致性 | 对话是否全部符合语言风格？ | 检查对话是否符合语言风格表 |
| 5 | 过审约束一致性 | 是否遵守过审约束？ | 检查是否使用架空表述、玄幻表述 |
| 6 | 参照物一致性 | 参照物与正式表述是否区分？ | 检查灵感/调研阶段是否使用参照物标签 |

**检查项**：
- [ ] 是否执行了六维度验证？
- [ ] 是否全部通过？
- [ ] 如果有未通过项，是否已修复？

## 八、快速检查清单（简化版）

> 当变更较简单时，可使用此简化清单快速检查。

### 8.1 B级变更快速检查

```
□ 变更是否只影响当前文档？
□ 是否不需要修改其他文档？
□ 是否不引入新的设定冲突？
□ 是否不增加过审风险？
→ 如果全部打勾，可以直接执行变更
```

### 8.2 A级变更快速检查

```
□ 是否已识别直接关联文档？
□ 是否已检查关系链断裂？
□ 是否已确定更新顺序？
□ 是否已准备变更记录？
→ 如果全部打勾，可以执行变更
```

### 8.3 S级变更完整检查

```
□ 是否已执行变更可行性评估？
□ 是否已按四层清单识别影响范围？
□ 是否已执行关系链断裂检测？
□ 是否已分析断裂点依赖关系？
□ 是否已评估修复方案？
□ 是否已选择更新顺序策略？
□ 是否已准备变更记录和回滚机制？
□ 是否已准备执行六维度验证？
→ 如果全部打勾，可以执行变更
```

## 附录：三次模拟场景总结

| # | 场景 | 方向 | 断裂点 | 受影响文档 | 流畅度 |
|---|------|------|--------|-----------|--------|
| 1 | 新增觉醒失控机制 | 顺向 | 0个 | 12个 | ★★★★☆ |
| 2 | 新增血脉共鸣设定 | 顺向 | 4个 | 17个 | ★★★★☆ |
| 3 | 新增归墟宗门设定 | 逆向 | 5个 | 22个 | ★★★☆☆ |

**核心发现**：
- 逆向传播比顺向传播困难一个量级
- 需要先评估底层能否支撑，再决定是否执行变更
- 需要回溯修改已完成阶段，并重新执行门控检查
