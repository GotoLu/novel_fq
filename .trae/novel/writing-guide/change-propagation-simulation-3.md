# 变更传播机制验证：大纲变更导致世界观底层逻辑冲突

## 场景设定

**触发阶段**：大纲规划阶段（二稿修改时调整大纲）
**变更内容**：新增"天机阁主是林墨白父亲的同门师兄弟"设定，引入"隐世宗门——归墟"概念
**变更原因**：大纲中天机阁主的动机不够充分——他为什么要背叛觉醒者？为什么要清洗同类？需要一个更有说服力的前史

**关键问题**：这个看似只是"人物关系加个前史"的变更，会直接导致世界观底层逻辑崩塌——

---

### 冲突分析

当前世界观设定中：
- 觉醒传承方式：家族传承（父子相传）、自然觉醒、血脉突破封印
- 觉醒历史：商朝巫师鼎盛 → 周朝封印 → 觉醒者转入地下 → 血脉逐渐稀薄
- 觉醒触发条件：绝境、强烈情感刺激、血脉浓度临界
- 社会规则：觉醒者被追杀，无组织，各自为战

**新增"归墟宗门"后的逻辑冲突**：

1. **传承体系冲突**：
   - 当前设定：觉醒传承只有"家族传承"和"自然觉醒"，没有系统训练
   - 新设定：存在一个隐世宗门"归墟"，能系统培养觉醒者
   - **冲突点1**：如果有宗门系统训练，为什么林墨白的父亲不知道？为什么林墨白只能靠自然觉醒？

2. **历史设定冲突**：
   - 当前设定：商朝灭亡后，觉醒者传承断绝，只剩零星血脉
   - 新设定：归墟从商朝传承至今，从未断绝
   - **冲突点2**：如果归墟从未断绝，为什么历史上没有归墟的痕迹？为什么觉醒者看起来像是"各自为战"？

3. **能力体系冲突**：
   - 当前设定：觉醒触发条件是"绝境/刺激/自然觉醒"，没有系统训练方法
   - 新设定：归墟有系统的觉醒训练方法
   - **冲突点3**：如果有系统训练方法，为什么林墨白只能靠绝境觉醒？为什么玄机子不教他系统训练？

4. **社会规则冲突**：
   - 当前设定：觉醒者被追杀，无组织，各自为战
   - 新设定：归墟是一个有组织的觉醒者团体
   - **冲突点4**：如果有组织，为什么觉醒者看起来像是无组织的？归墟为什么不保护其他觉醒者？

5. **核心冲突结构冲突**：
   - 当前设定：核心冲突是"个人 vs 体制"，觉醒者是被压迫的个体
   - 新设定：归墟是一个觉醒者组织，意味着觉醒者也有"体制"
   - **冲突点5**：核心冲突从"个人 vs 体制"变成"体制 vs 体制"，故事性质发生根本变化

### 第四步：逐断裂点修复

#### 4.1 修复断裂点5：核心冲突结构冲突

**问题**：归墟的引入将核心冲突从"个人 vs 体制"变成"体制 vs 体制"

**修复方案评估**：

| 方案 | 逻辑自洽 | 戏剧张力 | 角色深度 | 过审风险 | 修改成本 |
|------|---------|---------|---------|---------|---------|
| A：归墟是衰落的宗门，已无力组织 | ★★★★ | ★★☆ | ★★☆ | ★★★★★ | ★★☆ |
| B：归墟是秘密宗门，刻意隐藏 | ★★★ | ★★★★ | ★★★★ | ★★★★★ | ★★★☆ |
| C：归墟已分裂，守护派和清洗派各自为政 | ★★★★★ | ★★★★★ | ★★★★★ | ★★★★★ | ★★★★ |

**选择方案C**：归墟已分裂

**理由**：
- 逻辑最自洽：解释了为什么归墟不保护其他觉醒者（分裂后无力保护）
- 戏剧张力最强：守护派 vs 清洗派 = 觉醒者内部的思想对立
- 角色深度最深：天机阁主不是简单的叛徒，而是"清洗派"的代表
- 核心冲突从"个人 vs 体制"升级为"个人 + 守护派 vs 清洗派 + 朝廷"，层次更丰富

**关键决策**：核心冲突不变，仍然是"个人 vs 体制"，但增加了"觉醒者内部的思想对立"这一维度。归墟的分裂使得觉醒者无法形成统一力量，反而加深了"个人 vs 体制"的困境。

#### 4.2 修复断裂点1：传承体系冲突

**问题**：如果归墟有系统训练方法，林墨白为什么只能靠绝境觉醒？

**修复方案**：

```
归墟传承规则：
- 归墟的传承方式是"师徒制"，不是"家族制"
- 林墨白的父亲是归墟弟子，但他选择了"脱离归墟"
- 脱离原因：他不愿让儿子承受觉醒的痛苦和危险
- 他选择压制血脉，过普通人的生活
- 因此他没有把归墟的训练方法传给林墨白

关键设定：
- 归墟的传承有"入门试炼"，只有通过试炼才能学习训练方法
- 林墨白的父亲没有让林墨白参加试炼
- 林墨白的绝境觉醒，实际上就是归墟的"入门试炼"——在生死关头觉醒
- 玄机子后来收林墨白为徒，实际上是在引导他走归墟的训练路径
```

#### 4.3 修复断裂点2：历史设定冲突

**问题**：如果归墟从未断绝，为什么历史上没有痕迹？

**修复方案**：

```

  - 不与任何朝廷势力公开接触
  - 弟子以普通身份生活在社会中
- 归墟的衰落：
  - 历代天灾人祸，弟子越来越少
  - 清洗派与守护派的分裂加速了衰落
  - 到天启朝，归墟只剩下不到十人
- 归墟为什么不保护其他觉醒者：
  - 分裂后，守护派和清洗派互相牵制
  - 守护派想保护，但力量不足
  - 清洗派反而利用朝廷来清除"不纯"的觉醒者
```

#### 4.4 修复断裂点3：能力体系冲突

**问题**：如果有系统训练方法，为什么玄机子不教林墨白？

**修复方案**：

```

- 归墟的训练方法不是"教你觉醒"，而是"引导你控制觉醒"
- 觉醒仍然需要"触发条件"（绝境/刺激/血脉临界）
- 归墟的训练是在觉醒后，教你如何控制和精进能力
- 玄机子就是归墟守护派的最后传人
- 他收林墨白为徒，就是在教他归墟的训练方法
- 只是他没有告诉林墨白"归墟"的存在（因为归墟的秘密性）

能力体系调整：
- 觉醒触发：不变（绝境/刺激/血脉临界）
- 觉醒后训练：新增"归墟训练法"路径
  - 自然摸索：缓慢，容易失控
  - 归墟训练：系统，安全，但需要师父指导
- 林墨白的成长路径：自然觉醒 → 玄机子用归墟方法引导 → 逐步精进
```

### ⚠️ 新发现的流程卡点

#### 卡点9：更新顺序规则不适合逆向传播

**问题**：当前 outline-planning 的更新顺序规则是：
```
1. 先更新大纲自身
2. 再更新受影响的人物设定
3. 再更新受影响的世界观设定
4. 最后更新写作指南
```

但本次变更的5个断裂点中，4个是世界观层面的冲突。如果先更新大纲，再更新世界观，可能会发现世界观无法支撑大纲的变更，导致大纲需要再次修改——**返工**。

**根本原因**：当前更新顺序规则是"自上而下"的（先更新当前阶段，再更新下游阶段），但逆向传播场景需要"自下而上"（先确认底层设定能否支撑，再更新上层大纲）。

**建议**：在变更传播机制中增加"更新顺序策略选择"：

```
更新顺序策略：
- 顺向传播（世界观→人物→大纲→写作）：当前阶段是变更源头，按当前规则执行
- 逆向传播（大纲→人物→世界观）：当前阶段不是变更的底层支撑
  → 先评估底层能否支撑变更
  → 底层可以支撑：按当前规则执行
  → 底层无法支撑：先修复底层，再更新当前阶段
```

#### 卡点10：变更可能被否决——流程缺乏"变更可行性评估"步骤

**问题**：本次变更如果断裂点5（核心冲突结构冲突）的修复方案不可行，整个变更就需要被否决。但当前流程没有"变更可行性评估"步骤——它默认所有变更都可以被修复。

**示例**：如果作者提出的变更是"把故事从玄幻改为科幻"，这个变更会导致整个世界观崩塌，无法修复。但当前流程没有机制来识别和否决这种变更。

**建议**：在变更传播机制中增加"变更可行性评估"步骤（在执行更新之前）：

```
变更可行性评估：
1. 变更是否与核心主题冲突？（如果冲突，是否可以调和？）
2. 变更是否需要重写超过50%的文档？（如果是，考虑是否值得）
3. 变更是否改变故事的根本性质？（如果是，需要重新评估）
4. 变更的修复成本是否远大于收益？（如果是，考虑替代方案）

评估结果：
- 可行：按流程执行
- 有条件可行：修改变更方案后执行
- 不可行：否决变更，寻找替代方案
```

#### 卡点11：大纲变更可能需要"回溯"到已完成的阶段

**问题**：本次变更需要修改世界观设定（history.md、special-systems.md等），但世界观构建阶段已经"通过门控"完成了。当前流程没有"回溯到已完成阶段"的机制。

**根本原因**：阶段门控机制的设计是"单向"的——通过门控后进入下一阶段，不会回退。但实际创作中，后期阶段发现的问题经常需要回溯到前期阶段修改。

**建议**：在变更传播机制中增加"回溯修改"规则：

```
回溯修改规则：
1. 变更需要修改已完成阶段的文档时，必须：
   a. 标记回溯原因和范围
   b. 修改完成后重新执行该阶段的门控检查
   c. 确认修改不会导致该阶段其他文档不一致
2. 回溯修改的范围控制：
   - 只修改受影响的文档，不做"顺便"修改
   - 修改后必须更新变更记录
3. 回溯修改后的连锁检查：
   - 从回溯阶段开始，逐阶段检查是否有新的不一致
   - 确认所有后续阶段的文档仍然一致
```

**核心发现**：逆向传播（大纲→世界观）比顺向传播（世界观→大纲）困难得多，原因有三：

1. **更新顺序冲突**：顺向传播按"当前→下游"执行即可，逆向传播需要先确认底层能否支撑
2. **变更可能不可行**：顺向传播的变更通常可以修复，逆向传播的变更可能被底层逻辑否决
3. **需要回溯已完成阶段**：顺向传播只需更新后续阶段，逆向传播需要回溯修改已完成阶段
