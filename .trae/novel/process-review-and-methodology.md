## 二、踩过的坑与教训

### 2.1 流程顺序问题

**坑1：跳过市场调研直接规划大纲**
- 现象：灵感捕捉后直接进入大纲规划
- 问题：缺乏市场分析，定位不清
- 教训：必须按skill链顺序执行，市场调研不可跳过

**坑2：过审风险评估后置**
- 现象：大纲规划完才补充过审风险评估
- 问题：大纲中使用了真实历史朝代名称，后期需要大量修改
- 教训：过审风险评估应在市场调研阶段完成，影响后续所有环节

### 2.2 一致性问题

**坑3：架空化不彻底**
- 现象：世界观和大纲更新为"天启朝"，但灵感和市场调研文件未同步
- 问题：质量评估时发现不一致
- 教训：架空化处理需要全链路同步，但需区分"参照物"和"正式表述"

**坑4：参照物与正式表述混淆**
- 现象：灵感/市场调研中的"明朝"是否需要更新？
- 问题：最初认为需要全部替换，后来意识到参照物是合理的
- 教训：区分"创作参照物"（灵感/调研阶段）和"正式表述"（大纲/世界观/正文阶段）

### 2.3 质量控制问题

**坑5：缺乏阶段间质量门控**
- 现象：各阶段产出后没有质量检查就直接进入下一阶段
- 问题：问题积累到后期才发现，修改成本高
- 教训：每个阶段完成后应进行质量门控检查

**坑6：缺乏跨阶段一致性检查**
- 现象：世界观更新后，大纲未同步更新
- 问题：各阶段文档之间出现不一致
- 教训：任何阶段的关键变更都应触发跨阶段一致性检查

## 三、可沉淀的通用方法论

### 3.1 阶段门控机制（新增）

**问题**：缺乏阶段间质量门控，问题积累到后期才发现

**通用方法论**：每个阶段完成后，必须通过质量门控才能进入下一阶段

**门控检查项**：
1. 本阶段产出是否完整？
2. 本阶段产出是否与前一阶段一致？
3. 本阶段产出是否为下一阶段提供了充分输入？
4. 是否存在需要回溯修改的问题？

**沉淀位置**：所有skill文档的"约束条件"部分

### 3.2 参照物管理机制（新增）

**问题**：灵感/调研阶段的"明朝"参照物与正式表述混淆

**通用方法论**：区分"创作参照物"和"正式表述"，建立参照物管理机制

**参照物规则**：
- 灵感捕捉阶段：可以使用真实历史/现实作为参照物
- 市场调研阶段：竞品名称、市场定位可使用真实名称
- 大纲规划阶段：必须使用架空/虚构表述
- 世界观构建阶段：必须使用架空/虚构表述
- 初稿写作阶段：必须使用架空/虚构表述

**沉淀位置**：inspiration-capture、market-research的skill文档

### 3.3 架空化处理流程（新增）

**问题**：架空化不彻底，后期需要大量修改

**通用方法论**：架空化处理应有标准流程，确保全链路同步

**架空化流程**：
1. 确定架空化范围（朝代、地名、人名、机构名等）
2. 设计架空名称（至少5个备选，详细说明依据）
3. 选定最终名称
4. 更新所有正式文档（大纲、世界观、正文）
5. 保留参照物文档（灵感、调研）
6. 建立架空名称对照表

**沉淀位置**：world-building的skill文档

### 3.4 变更传播机制（新增）

**问题**：世界观更新后，大纲未同步更新

**通用方法论**：任何阶段的关键变更都应触发变更传播检查

**变更传播规则**：
1. 识别变更影响范围
2. 列出需要同步更新的文档
3. 逐一更新受影响的文档
4. 验证更新后的跨阶段一致性

**沉淀位置**：所有skill文档的"约束条件"部分

### 3.5 过审风险前置评估（已有，需强化）

**问题**：过审风险评估后置，导致大纲需要大量修改

**通用方法论**：过审风险评估必须在市场调研阶段完成

**强化要点**：
- 过审风险评估是市场调研的必要环节
- 过审约束条件必须在市场调研阶段确定
- 后续所有阶段必须遵守过审约束

**沉淀位置**：market-research的skill文档（已实现）

### 3.6 语言风格体系（新增）

**问题**：初稿写作时缺乏语言风格参考

**通用方法论**：人物设计阶段必须产出语言风格表

**语言风格体系**：
1. 每个主要人物必须有语言风格定义
2. 语言风格必须随人物成长演变
3. 必须包含典型对话示例
4. 必须包含正确/错误示例对比

**沉淀位置**：character-design的skill文档

### 3.7 章节自检体系（已有，需强化）

**问题**：初稿写作缺乏质量检查机制

**通用方法论**：每章写作完成后必须进行自检

**强化要点**：
- 15项核心自检要点
- 对话专项检查
- 自检记录必须存档

**沉淀位置**：first-draft-writing的skill文档（已实现）

### 3.8 质量评估机制（新增）

**问题**：缺乏整体质量评估

**通用方法论**：所有阶段完成后，进行整体质量评估

**质量评估要点**：
1. 各阶段产出完整性
2. 跨阶段一致性
3. 过审约束遵循
4. 改进建议和优先级

**沉淀位置**：新增quality-assessment skill或作为通用机制

## 四、skill链改进建议

### 4.1 现有skill链

```
灵感捕捉 → 市场调研 → 大纲规划 → 世界观构建 → 人物设计 → 初稿写作 → 二稿修改 → 三稿精修 → 专业审稿 → 定稿
```

### 4.2 改进后的skill链

```
灵感捕捉 → [门控检查] → 市场调研（含过审风险评估）→ [门控检查] → 大纲规划 → [门控检查] → 世界观构建（含架空化处理）→ [门控检查+变更传播] → 人物设计（含语言风格体系）→ [门控检查] → 初稿写作（含章节自检）→ 二稿修改 → 三稿精修 → 专业审稿 → 定稿
```

### 4.3 改进要点

**1. 门控检查机制**：
- 每个阶段完成后，必须通过门控检查
- 门控检查包括：完整性、一致性、充分性、回溯性
- 门控检查不通过，不得进入下一阶段

**2. 过审风险评估前置**：
- 过审风险评估在市场调研阶段完成
- 过审约束条件影响后续所有阶段
- 后续阶段必须遵守过审约束

**3. 架空化处理流程**：
- 世界观构建阶段包含架空化处理
- 架空化处理有标准流程
- 架空化处理触发变更传播

**4. 变更传播机制**：
- 任何阶段的关键变更触发变更传播
- 变更传播确保跨阶段一致性
- 变更传播有检查和验证

**5. 语言风格体系**：
- 人物设计阶段必须产出语言风格表
- 语言风格表包含演变轨迹和示例
- 语言风格表是初稿写作的必要参考

**6. 章节自检体系**：
- 初稿写作阶段必须进行章节自检
- 自检包括15项核心要点
- 自检记录必须存档

## 六、通用方法论总结

### 6.1 核心方法论

| 方法论 | 解决的问题 | 适用阶段 |
|--------|-----------|----------|
| 阶段门控机制 | 问题积累到后期才发现 | 所有阶段 |
| 参照物管理机制 | 参照物与正式表述混淆 | 灵感、调研 |
| 架空化处理流程 | 架空化不彻底 | 世界观构建 |
| 变更传播机制 | 跨阶段不一致 | 所有阶段 |
| 过审风险前置评估 | 过审风险后置 | 市场调研 |
| 语言风格体系 | 初稿缺乏语言参考 | 人物设计 |
| 章节自检体系 | 初稿缺乏质量检查 | 初稿写作 |
| 质量评估机制 | 缺乏整体质量评估 | 所有阶段完成后 |

### 6.2 核心原则

1. **顺序不可跳过**：skill链的顺序是有逻辑的，不可跳过任何阶段
2. **门控不可省略**：每个阶段完成后必须通过门控检查
3. **约束必须遵守**：过审约束等约束条件必须遵守
4. **变更必须传播**：关键变更必须触发变更传播
5. **参照物需区分**：创作参照物和正式表述必须区分
6. **自检必须执行**：每章写作完成后必须进行自检
7. **记录必须存档**：所有检查记录必须存档

### 6.3 预期效果

**减少返工**：
- 门控机制避免问题积累
- 变更传播避免跨阶段不一致
- 过审前置避免后期大量修改

**提升质量**：
- 自检体系确保每章质量
- 语言风格体系确保对话质量
- 质量评估确保整体质量

**提高效率**：
- 标准流程减少决策时间
- 参照物管理减少混淆
- 架空化流程减少遗漏
