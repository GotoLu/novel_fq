## 执行模式选择

### 模式1：顺序执行（推荐新手）

严格按照技能链顺序依次执行，每个技能完成后再进入下一个。

**优点**：
- 流程清晰，不易遗漏
- 每个阶段都能充分准备
- 适合新手学习和理解

**缺点**：
- 耗时较长
- 无法并行优化

---

### 模式2：迭代执行（推荐有经验者）

在某个阶段反复迭代，直到达到质量标准后再进入下一阶段。

**优点**：
- 质量更有保障
- 可以不断优化
- 适合追求高质量的创作者

**缺点**：
- 可能陷入过度修改
- 时间不可控

**实施方法**：
```markdown
迭代循环：
1. 执行技能
2. 质量评估
3. 是否达标？
   - 是 → 进入下一技能
   - 否 → 继续迭代本技能
```

---

### 模式3：并行执行（推荐团队协作）

第二阶段的"人物设计"和"世界观构建"并行执行。

**优点**：
- 节省时间
- 提高效率
- 适合团队分工

**缺点**：
- 需要协调沟通
- 可能出现不一致

**实施方法**：
```markdown
并行执行：
1. 团队成员A负责人物设计
2. 团队成员B负责世界观构建
3. 定期同步进度
4. 完成后合并输出
5. 进行一致性检查
```

## 回溯与调整

### 何时需要回溯

- 后续阶段发现前置阶段的重大问题
- 人物行为与设定严重不符
- 世界观设定出现矛盾
- 情节发展偏离大纲

### 回溯流程

```
发现问题
    ↓
评估影响范围
    ↓
确定回溯目标技能
    ↓
创建当前版本备份
    ↓
回溯修改
    ↓
重新执行受影响的后续技能
    ↓
验证一致性
```

### 回溯注意事项

1. **评估影响**：修改前评估对后续所有技能的影响
2. **版本管理**：每次回溯都创建新版本
3. **记录原因**：详细记录回溯的原因和修改内容
4. **重新验证**：修改后重新执行所有受影响的技能
5. **保持一致**：确保修改后的整体一致性

## 常见问题与解决方案

### Q1：灵感不足怎么办？

**解决方案**：
- 扩大灵感来源（阅读、观影、旅行、交流）
- 使用灵感激发技巧（头脑风暴、思维导图）
- 暂时跳过，先进行市场调研寻找方向

### Q2：大纲规划时卡住了怎么办？

**解决方案**：
- 回顾核心创意，找回初心
- 参考经典作品的结构设计
- 先制定粗略大纲，后续迭代细化
- 与他人讨论，获取外部视角

### Q3：初稿写作进度缓慢怎么办？

**解决方案**：
- 降低每日目标，保持持续性
- 遇到卡点先标记，继续前进
- 调整写作时间和环境
- 寻找写作伙伴互相督促

### Q4：修改时越改越乱怎么办？

**解决方案**：
- 停止修改，休息几天
- 回到大纲，找回主线
- 聚焦一类问题，分轮修改
- 寻求外部反馈，获得客观视角

### Q5：不知道何时可以定稿怎么办？

**解决方案**：
- 对照质量标准检查
- 组织试读获取反馈
- 咨询专业人士意见
- 设定截止日期，避免无限修改

## 最佳实践建议

### 1. 文档管理

- 使用统一的文件命名规范
- 定期备份所有文档
- 使用版本控制系统
- 保持文档结构清晰

### 2. 时间管理

- 制定详细的创作计划
- 设定里程碑和截止日期
- 预留缓冲时间
- 定期回顾和调整计划

### 3. 质量管理

- 严格执行检查点验证
- 不跳过任何质量检查
- 及时记录和修正问题
- 保持高标准不妥协

### 4. 自我管理

- 保持创作节奏
- 适度休息避免疲劳
- 保持创作热情
- 庆祝每个里程碑达成

### 5. 持续改进

- 记录创作经验教训
- 优化个人工作流程
- 学习新的创作技巧
- 反思和总结每个项目

## 技能链扩展

### 添加新技能

如需添加新技能到技能链：

1. 确定技能的定位和作用
2. 明确输入输出关系
3. 定义质量标准
4. 更新技能链配置
5. 测试和验证

### 定制个人技能链

根据个人创作习惯定制：

1. 评估现有技能链的适用性
2. 识别需要调整的部分
3. 增删或修改技能
4. 调整执行顺序
5. 验证定制后的效果
