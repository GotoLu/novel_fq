# 大纲规划技能

## 核心能力描述

大纲规划是小说创作的蓝图设计阶段，该技能帮助创作者构建完整的故事结构、规划情节发展、设计节奏控制，为后续写作提供清晰的路线图。

## 关键操作步骤

### 1. 主题确定
- **核心主题提炼**：
  - 从灵感中提炼核心思想
  - 确定主题的普世性和深度
  - 主题的多层次表达（表层/深层）
- **主题呈现方式**：
  - 显性表达 vs 隐性渗透
  - 通过人物命运体现
  - 通过情节冲突展现
  - 通过象征意象暗示
- **主题与时代**：主题的时代意义和现实关照

### 2. 核心冲突设计
- **冲突类型选择**：
  - 人与自然（生存挑战）
  - 人与社会（制度、观念、利益冲突）
  - 人与人（情感、利益、价值观冲突）
  - 人与自我（内心挣扎、成长蜕变）
- **冲突层次**：
  - 外部冲突（表层情节推动）
  - 内部冲突（人物内心矛盾）
  - 价值观冲突（深层思想碰撞）
- **冲突升级路径**：
  - 初始冲突设定
  - 冲突逐步升级
  - 冲突高潮设计
  - 冲突解决方式

### 3. 故事结构设计
- **三幕式结构**：
  - 第一幕：铺垫（占20-25%）
    - 开场钩子
    - 人物介绍
    - 世界观呈现
    - 激励事件
  - 第二幕：对抗（占50-60%）
    - 冲突发展
    - 情节转折
    - 中点高潮
    - 危机加深
  - 第三幕：解决（占20-25%）
    - 最终高潮
    - 冲突解决
    - 人物成长
    - 结局收尾

- **其他结构模式**：
  - 五幕式结构（铺垫-发展-高潮-回落-结局）
  - 多线叙事结构
  - 倒叙/插叙结构
  - 环形结构
  - 碎片化结构

### 4. 章节大纲制定
- **章节数量规划**：
  - 根据总字数确定章节数
  - 每章字数建议（3000-8000字）
  - 卷/部的划分
- **章节内容设计**：
  - 每章核心事件
  - 每章人物行动
  - 每章情感变化
  - 每章推进目标
- **章节节奏控制**：
  - 高潮章节分布
  - 过渡章节安排
  - 悬念设置节点
  - 信息释放节奏
- **章节钩子**：
  - 每章结尾钩子设计
  - 章节间衔接方式

### 5. 情节线索规划
- **主线情节**：
  - 主线发展脉络
  - 主线关键节点
  - 主线推进节奏
- **支线情节**：
  - 支线数量控制（2-5条）
  - 支线与主线关系
  - 支线交织节点
  - 支线收束方式
- **伏笔设计**：
  - 伏笔埋设位置
  - 伏笔回收节点
  - 伏笔隐蔽性控制

### 6. 字数与节奏规划
- **总字数确定**：
  - 根据题材和读者定位
  - 短篇：1-3万字
  - 中篇：3-10万字
  - 长篇：10-50万字
  - 超长篇/网文：100万字以上
- **节奏控制**：
  - 开篇节奏（快速吸引）
  - 发展节奏（张弛有度）
  - 高潮节奏（紧凑激烈）
  - 结尾节奏（从容收尾）
- **信息密度**：
  - 情节密度控制
  - 信息释放节奏
  - 留白与充实平衡

### 7. 关键场景设计
- **开场设计**：
  - 开场方式（悬念/冲突/氛围/人物）
  - 开场钩子强度
  - 开场信息量控制
- **高潮设计**：
  - 主要高潮场景（2-3个）
  - 高潮前铺垫
  - 高潮后缓冲
- **转折设计**：
  - 情节转折点（3-5个）
  - 转折合理性
  - 转折冲击力
- **结局设计**：
  - 结局类型（圆满/开放/悲剧/反转）
  - 结局合理性
  - 结局余味

## 所需知识储备

### 故事理论

- 叙事学和情节理论
- 戏剧冲突理论
- 类型小说创作规律

### 文学素养
- 中外经典作品结构分析
- 不同题材的结构特点
- 叙事节奏控制技巧
- 悬念和伏笔运用

### 逻辑思维
- 因果关系构建
- 情节合理性推演
- 人物行为逻辑
- 时间线管理

### 工具使用
- 大纲编写软件（Scrivener、yWriter等）

## 评估标准

### 定量指标
- **结构完整性**：包含开篇、发展、高潮、结局四要素
- **章节数合理**：总字数/章节平均字数 = 章节数
- **节奏分布**：高潮场景占比15-25%
- **伏笔回收率**：埋设伏笔回收率 ≥ 95%

### 定性指标
- **逻辑自洽**：情节发展符合因果逻辑
- **冲突强度**：核心冲突具有足够张力和深度
- **节奏合理**：张弛有度，无冗长或仓促感
- **主题清晰**：主题贯穿始终，表达明确
- **创新性**：结构有新意，不落俗套

### 能力等级
- **初级**：能制定基本大纲，但结构较简单
- **中级**：能设计完整结构，节奏控制合理
- **高级**：结构精巧，节奏完美，冲突设计深刻
- **专家级**：能创新结构模式，引领叙事革新

## 实践建议

1. **参考经典**：学习经典作品的结构设计
2. **反复推敲**：大纲需要多次修改和完善
3. **预留弹性**：大纲是指导而非束缚，保持调整空间
4. **验证逻辑**：检查情节发展的合理性和必然性
5. **节奏测试**：想象阅读体验，测试节奏是否舒适
6. **工具辅助**：使用大纲软件提高效率

## 约束条件

### 过审约束遵守

**大纲规划必须遵守市场调研阶段确定的过审约束**：
1. 朝代表述：必须使用架空/虚构朝代名称，禁止真实历史朝代名称
2. 能力表述：必须使用玄幻表述，禁止迷信敏感词汇
3. 历史处理：完全架空，不涉及真实历史事件
4. 价值观：积极向上，强调成长和责任
5. 内容把控：避免过度暴力、血腥、政治隐喻

### 变更传播机制

**大纲规划阶段发生关键变更时，必须触发变更传播检查**：

1. **识别变更影响范围**：哪些设定发生了变化？
2. **列出受影响文档**：哪些后续阶段的文档需要同步更新？
3. **逐一更新**：按清单更新所有受影响的文档
4. **验证一致性**：确认更新后的跨阶段一致性

**变更分级**：
- **S级变更**（核心设定变更）：全链路更新，如核心冲突、主题方向变更
- **A级变更**（重要设定调整）：更新直接关联文档，如支线情节调整
- **B级变更**（细节修改）：只更新当前文档，如章节标题修改

**影响范围识别清单**（按层级排查）：
- 第一层：直接关联（章节大纲 → 冲突地图 → 关键场景 → 伏笔设计）
- 第二层：人物关联（主角小传 → 反派小传 → 配角小传 → 人物关系）
- 第三层：世界观关联（能力体系 → 社会规则 → 历史设定）
- 第四层：写作关联（语言风格 → 自检清单 → 对话示例）

**更新顺序规则**：
1. 先更新大纲自身（章节 → 冲突 → 场景 → 伏笔）
2. 再更新受影响的人物设定
3. 再更新受影响的世界观设定
4. 最后更新写作指南

**变更记录模板**：
```
变更编号：CP-[阶段缩写]-[序号]
变更日期：YYYY-MM-DD
触发阶段：大纲规划
变更内容：[具体描述]
变更原因：[为什么需要变更]
变更级别：S/A/B级
受影响文档：[列表]
更新状态：进行中/已完成
验证结果：通过/未通过
```

**变更回滚机制**：
- 变更前备份当前版本
- 变更过程中如发现方向不对，可回滚到变更前版本
- 变更完成后验证不通过，也可回滚

**关系链断裂检测**（S/A级变更必须执行）：
1. 新设定是否与现有角色身份冲突？（每个角色的身份/职业/知识是否与新设定矛盾）
2. 新设定是否与现有角色关系冲突？（角色之间的信息差、互动是否与新设定矛盾）
3. 新设定是否与现有情节冲突？（已发生的情节、角色的行为是否与新设定矛盾）

**修复方案评估**（发现断裂点后）：
1. 逻辑自洽性：修复方案是否与现有设定逻辑自洽？
2. 戏剧张力：修复方案是否增加了戏剧张力？
3. 角色深度：修复方案是否深化了角色？
4. 过审风险：修复方案是否增加过审风险？
5. 修改成本：修复方案需要修改多少文档？

**断裂点依赖分析**（多个断裂点时）：
1. 列出所有断裂点
2. 分析断裂点之间的依赖关系
3. 确定修复顺序（先修复上游断裂点）
4. 上游修复方案确定后，再评估下游断裂点

**更新顺序策略选择**：
- 顺向传播（世界观→人物→大纲→写作）：变更源头在上游，按当前规则"先更新自身→再更新下游"执行
- 逆向传播（大纲→人物→世界观）：变更源头在下游，需先评估底层能否支撑变更
  - 底层可以支撑：按当前规则执行
  - 底层无法支撑：先修复底层，再更新当前阶段

**变更可行性评估**（S级变更必须执行，在更新之前）：
1. 变更是否与核心主题冲突？（如果冲突，是否可以调和？）
2. 变更是否需要重写超过50%的文档？（如果是，考虑是否值得）
3. 变更是否改变故事的根本性质？（如果是，需要重新评估）
4. 变更的修复成本是否远大于收益？（如果是，考虑替代方案）
评估结果：可行 / 有条件可行 / 不可行（否决变更，寻找替代方案）

**回溯修改规则**（变更需要修改已完成阶段的文档时）：
1. 标记回溯原因和范围
2. 修改完成后重新执行该阶段的门控检查
3. 确认修改不会导致该阶段其他文档不一致
4. 只修改受影响的文档，不做"顺便"修改
5. 修改后必须更新变更记录
6. 从回溯阶段开始，逐阶段检查是否有新的不一致

**常见触发场景**：
- 核心冲突设计变更
- 主要人物设定变更
- 世界观关键设定变更
- 过审约束调整

### 阶段门控

**大纲规划完成后，必须通过以下门控检查才能进入下一阶段**：

1. **完整性检查**：故事结构、章节大纲、冲突设计是否完整？
2. **逻辑检查**：情节发展是否逻辑自洽？
3. **过审检查**：大纲是否遵守过审约束？
4. **一致性检查**：大纲是否与灵感方向和市场定位一致？
5. **可执行检查**：大纲是否为世界观构建和人物设计提供了充分输入？

**门控不通过则**：修正大纲，直到通过门控检查。

## 输出成果

- 故事结构图（整体框架可视化）
- 章节大纲表（每章核心内容）
- 情节时间线（事件顺序和时长）
- 冲突发展图（冲突升级路径）
- 关键场景清单（重要场景设计）
- 节奏规划表（高潮和过渡分布）
- 伏笔设计表（埋设和回收计划）
