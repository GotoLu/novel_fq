# 小说创作技能链

## 技能链概述

本技能链将小说创作的10个核心技能按照逻辑顺序和依赖关系进行系统性整合，形成一条从灵感到出版的完整创作路径。每个技能阶段都有明确的输入输出，前一阶段的成果为后一阶段提供基础，确保创作过程的连贯性和系统性。

## 技能链详细说明

### 📍 第一阶段：创意与构思（基础构建）

#### 技能顺序：灵感捕捉 → 市场调研 → 大纲规划

#### 1. 灵感捕捉 (inspiration-capture)
- **阶段定位**：创作起点
- **前置条件**：无
- **输入**：生活观察、梦境、新闻、阅读、情感体验
- **输出**：
  - 灵感库（结构化记录）
  - 核心创意清单（优先级排序）
  - 创意概念说明
- **交付物**：`.trae/novel/inspiration/` 目录
  - `inspiration-library.md` - 灵感库
  - `core-ideas.md` - 核心创意清单
  - `concept-notes.md` - 创意概念说明

#### 2. 市场调研 (market-research)
- **阶段定位**：方向确定
- **前置条件**：灵感捕捉完成，有核心创意
- **输入**：
  - 核心创意清单（来自灵感捕捉）
  - 市场数据（外部获取）
- **输出**：
  - 市场调研报告
  - 目标读者画像
  - 竞品分析报告
  - 作品定位方案
- **交付物**：`.trae/novel/market-research/` 目录
  - `market-report.md` - 市场调研报告
  - `target-readers.md` - 目标读者画像
  - `competitor-analysis.md` - 竞品分析
  - `positioning.md` - 作品定位方案

#### 3. 大纲规划 (outline-planning)
- **阶段定位**：蓝图设计
- **前置条件**：灵感捕捉 + 市场调研完成
- **输入**：
  - 核心创意（来自灵感捕捉）
  - 作品定位（来自市场调研）
- **输出**：
  - 故事结构图
  - 章节大纲表
  - 情节时间线
  - 冲突发展图
  - 关键场景清单
  - 节奏规划表
  - 伏笔设计表
- **交付物**：`.trae/novel/outline/` 目录
  - `structure.md` - 故事结构
  - `chapter-outline.md` - 章节大纲
  - `timeline.md` - 情节时间线
  - `conflict-map.md` - 冲突发展图
  - `key-scenes.md` - 关键场景
  - `pacing-plan.md` - 节奏规划
  - `foreshadowing.md` - 伏笔设计

---

### 📍 第二阶段：人物与世界观（创作基础）

#### 技能顺序：人物设计 ∥ 世界观构建（可并行）

#### 4. 人物设计 (character-design)
- **阶段定位**：角色塑造
- **前置条件**：大纲规划完成
- **输入**：
  - 章节大纲（来自大纲规划）
  - 冲突发展图（来自大纲规划）
- **输出**：
  - 人物小传集
  - 人物关系图谱
  - 人物成长弧线图
  - 人物行为指南
  - 人物语言风格表
- **交付物**：`.trae/novel/characters/` 目录
  - `character-bios/` - 人物小传集
  - `relationship-map.md` - 人物关系图谱
  - `growth-arcs.md` - 成长弧线
  - `behavior-guide.md` - 行为指南
  - `language-styles.md` - 语言风格

#### 5. 世界观构建 (world-building)
- **阶段定位**：舞台搭建
- **前置条件**：大纲规划完成
- **输入**：
  - 章节大纲（来自大纲规划）
  - 作品定位（来自市场调研）
- **输出**：
  - 世界观总纲
  - 地理环境设定
  - 社会规则手册
  - 文化体系文档
  - 特殊体系设定
  - 世界观呈现计划
  - 设定查询手册
- **交付物**：`.trae/novel/worldbuilding/` 目录
  - `world-overview.md` - 世界观总纲
  - `geography.md` - 地理环境
  - `social-rules.md` - 社会规则
  - `culture.md` - 文化体系
  - `special-systems.md` - 特殊体系
  - `presentation-plan.md` - 呈现计划
  - `reference.md` - 设定查询

**第二阶段整合输出**：
- 创作基础材料包（人物 + 世界观）
- 为初稿写作提供完整支撑

### 📍 第三阶段：写作阶段（核心创作）

#### 技能顺序：初稿写作 → 二稿修改 → 三稿精修

#### 6. 初稿写作 (first-draft-writing)
- **阶段定位**：文字落地
- **前置条件**：大纲规划 + 人物设计 + 世界观构建完成
- **输入**：
  - 章节大纲（来自大纲规划）
  - 人物小传（来自人物设计）
  - 世界观设定（来自世界观构建）
- **输出**：
  - 小说初稿全文
  - 写作日志
  - 问题标记清单
  - 创作心得总结
- **交付物**：`.trae/novel/drafts/` 目录
  - `draft-01-first.md` - 初稿全文
  - `writing-log.md` - 写作日志
  - `issues-marked.md` - 问题标记
  - `creation-notes.md` - 创作心得

#### 7. 二稿修改 (second-draft-revision)
- **阶段定位**：结构优化
- **前置条件**：初稿写作完成
- **输入**：
  - 初稿全文（来自初稿写作）
  - 问题标记清单（来自初稿写作）
  - 章节大纲（参考对照）
- **输出**：
  - 二稿全文
  - 修改记录文档
  - 问题清单
  - 结构调整说明
  - 待进一步优化事项
- **交付物**：`.trae/novel/drafts/` 目录
  - `draft-02-second.md` - 二稿全文
  - `revision-log-02.md` - 修改记录
  - `issues-fixed-02.md` - 已修正问题
  - `structure-changes.md` - 结构调整
  - `issues-pending.md` - 待优化事项

#### 8. 三稿精修 (third-draft-polishing)
- **阶段定位**：文字打磨
- **前置条件**：二稿修改完成
- **输入**：
  - 二稿全文（来自二稿修改）
  - 待优化事项（来自二稿修改）
- **输出**：
  - 三稿全文
  - 修改记录
  - 风格说明
  - 质量评估
  - 待专业审稿事项
- **交付物**：`.trae/novel/drafts/` 目录
  - `draft-03-third.md` - 三稿全文
  - `revision-log-03.md` - 修改记录
  - `style-guide.md` - 风格说明
  - `quality-assessment.md` - 质量评估
  - `review-focus.md` - 审稿关注点

---

#### 技能顺序：专业审稿 → 定稿

#### 9. 专业审稿 (professional-review)
- **阶段定位**：质量把关
- **前置条件**：三稿精修完成
- **输入**：
  - 三稿全文（来自三稿精修）
  - 审稿关注点（来自三稿精修）
- **输出**：
  - 审稿后稿件
  - 问题清单
  - 试读反馈汇总
  - 修改记录
  - 质量评估报告
  - 待定稿事项
- **交付物**：`.trae/novel/review/` 目录
  - `draft-04-reviewed.md` - 审稿后稿件
  - `issues-final.md` - 最终问题清单
  - `reader-feedback.md` - 试读反馈
  - `revision-log-04.md` - 修改记录
  - `quality-report.md` - 质量评估
  - `final-checklist.md` - 待定稿事项

#### 10. 定稿 (final-manuscript)
- **阶段定位**：创作完成
- **前置条件**：专业审稿完成
- **输入**：
  - 审稿后稿件（来自专业审稿）
  - 待定稿事项（来自专业审稿）
- **输出**：
  - 最终定稿
  - 书名确定
  - 内容提要
  - 作者简介
  - 出版材料包
  - 版本说明
  - 备份记录
  - 出版计划
- **交付物**：`.trae/novel/final/` 目录
  - `manuscript-final.md` - 最终定稿
  - `book-title.md` - 书名
  - `synopsis.md` - 内容提要
  - `author-bio.md` - 作者简介

  - `version-info.md` - 版本说明
  - `backup-record.md` - 备份记录

## 技能链执行流程

### 标准执行顺序

```
1. 灵感捕捉
   ↓
2. 市场调研
   ↓
3. 大纲规划
   ↓
4. 人物设计 ──┐
              ├──→ (并行执行)
5. 世界观构建 ─┘
   ↓
6. 初稿写作
   ↓
7. 二稿修改
   ↓
8. 三稿精修
   ↓
9. 专业审稿
   ↓
10. 定稿
```

### 阶段性里程碑

| 里程碑 | 完成技能 | 标志性成果 |
|--------|---------|-----------|
| M1 | 灵感捕捉、市场调研、大纲规划 | 故事框架确定 |
| M2 | 人物设计、世界观构建 | 创作基础完备 |
| M3 | 初稿写作 | 初稿完成 |
| M4 | 二稿修改、三稿精修 | 精修稿完成 |
| M5 | 专业审稿、定稿 | 最终定稿完成 |

### 关键检查点

每个阶段完成后需要进行检查点验证：

**检查点1（M1）**：故事框架是否完整？
- ✅ 核心创意明确
- ✅ 市场定位清晰
- ✅ 大纲结构完整
- ✅ 章节规划合理

**检查点2（M2）**：创作基础是否完备？
- ✅ 主要人物小传完整
- ✅ 人物关系清晰
- ✅ 世界观逻辑自洽
- ✅ 设定支撑情节

**检查点3（M3）**：初稿是否达标？
- ✅ 完成预定字数
- ✅ 情节完整连贯
- ✅ 人物行为合理
- ✅ 无重大逻辑漏洞

**检查点4（M4）**：精修稿质量如何？
- ✅ 结构优化完成
- ✅ 人物刻画深入
- ✅ 语言精炼生动
- ✅ 风格统一协调

**检查点5（M5）**：是否可以定稿？
- ✅ 专业审稿通过
- ✅ 文字错误率达标
- ✅ 出版材料齐全
- ✅ 版本冻结确认

## 技能链使用指南

### 1. 初始化技能链

在项目根目录创建以下结构：

```
.trae/
└── novel/
    ├── inspiration/          # 灵感捕捉成果
    ├── market-research/      # 市场调研成果
    ├── outline/              # 大纲规划成果
    ├── characters/           # 人物设计成果
    ├── worldbuilding/        # 世界观构建成果
    ├── drafts/               # 各版本稿件
    ├── review/               # 审稿成果
    └── final/                # 最终定稿
```

### 2. 执行技能链

#### 顺序执行模式
按照技能链顺序依次执行每个技能，完成一个阶段后再进入下一阶段。

#### 迭代执行模式
允许在某个阶段反复迭代，直到达到该阶段的质量标准后再进入下一阶段。

#### 并行执行模式
第二阶段的"人物设计"和"世界观构建"可以并行执行，提高效率。

### 3. 技能链状态跟踪

使用 `skill-chain-status.md` 文件跟踪每个技能的执行状态：

```markdown
| 技能名称 | 状态 | 开始时间 | 完成时间 | 输出验证 |
|---------|------|---------|---------|---------|
| 灵感捕捉 | ✅ 完成 | 2024-01-01 | 2024-01-05 | ✅ 通过 |
| 市场调研 | ✅ 完成 | 2024-01-06 | 2024-01-10 | ✅ 通过 |
| 大纲规划 | 🔄 进行中 | 2024-01-11 | - | - |
| ... | ⏸️ 待开始 | - | - | - |
```

### 4. 技能链回溯与调整

- **允许回溯**：在后续阶段发现问题时，可以回溯到前面的技能进行修正
- **影响评估**：修改前评估对后续技能的影响范围
- **版本管理**：每次回溯修改都创建新版本，保留修改记录
- **重新验证**：修改后重新执行受影响的后续技能

## 技能链质量保障

### 输入验证
每个技能执行前验证输入是否完整和有效：
- 检查必需的输入文件是否存在
- 验证输入数据的完整性
- 确认前置技能已完成

### 输出验证
每个技能执行后验证输出是否达标：
- 检查所有输出文件是否生成
- 验证输出内容的质量标准
- 确认输出可作为后续技能的输入

### 一致性验证
定期验证整个技能链的一致性：
- 各阶段输出之间的逻辑一致性
- 人物设定的前后一致性
- 世界观设定的自洽性
- 情节发展的合理性

## 技能链优势

1. **系统性**：完整的创作流程覆盖，无遗漏环节
2. **连贯性**：各阶段环环相扣，逻辑清晰
3. **可追溯**：每个阶段的输出都有据可查
4. **可迭代**：支持反复迭代和优化
5. **可并行**：部分阶段可并行执行，提高效率
6. **可回溯**：允许回溯修改，保持灵活性
7. **质量保障**：每个阶段都有明确的质量标准
8. **文档化**：全程文档记录，便于复盘和改进

## 注意事项

1. **灵活应用**：技能链是指导框架，可根据实际情况调整
2. **避免教条**：不拘泥于固定顺序，根据创作需要灵活安排
3. **保持创造**：技能链服务于创作，不应束缚创造力
4. **持续迭代**：技能链本身也需要不断优化和完善
5. **个性定制**：根据个人创作习惯定制适合自己的技能链

## 扩展建议

### 技能链扩展方向

1. **增加子技能**：在每个主技能下增加细分技能

3. **增加协作技能**：支持团队协作的技能
4. **增加AI辅助技能**：AI辅助创作的技能
5. **增加出版后技能**：营销推广、读者运营等技能

### 技能链变体

- **短篇小说技能链**：简化版，适用于短篇创作
- **网文创作技能链**：针对网文特点的变体
- **系列小说技能链**：支持系列创作的扩展版
- **协作创作技能链**：支持团队协作的版本
