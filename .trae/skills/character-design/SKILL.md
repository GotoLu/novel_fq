# 人物设计技能

## 核心能力描述

人物设计是小说创作的核心环节，该技能帮助创作者塑造鲜活立体的人物形象、构建合理的人物关系网络、设计人物成长轨迹，使人物成为推动情节和承载主题的关键载体。

## 关键操作步骤

### 1. 主角设计
- **基本设定**：
  - 姓名（符合时代和身份）
  - 年龄、性别、职业
  - 外貌特征（独特标识）
  - 家庭背景和成长环境
- **性格塑造**：
  - 核心性格特质（2-3个主要特质）
  - 性格优点和缺点
  - 性格形成原因（成长经历）
  - 性格的复杂性和矛盾性
- **内在驱动**：
  - 核心欲望（最想要什么）
  - 核心恐惧（最怕什么）
  - 价值观和信念
  - 内心创伤或阴影
- **人物弧线**：
  - 起始状态（开篇时的人物状态）
  - 变化契机（促使改变的事件）
  - 成长过程（内在和外在的变化）
  - 终极状态（结局时的人物状态）
- **行为特征**：
  - 语言风格（口头禅、说话方式）
  - 行为习惯（标志性动作）
  - 决策模式（面对选择时的倾向）
  - 情绪表达方式

### 2. 配角设计
- **功能性分类**：
  - 导师型（指引主角）
  - 伙伴型（陪伴支持）
  - 对立型（制造冲突）
  - 衬托型（对比衬托）
  - 功能型（推动情节）
- **配角原则**：
  - 每个配角都有明确功能
  - 避免功能重复的配角
  - 配角也要有立体性
  - 配角数量控制（主要配角3-8人）
- **配角深度**：
  - 重要配角需完整人物小传
  - 次要配角可简化但要有特点
  - 龙套角色也要有记忆点

### 3. 反派设计
- **反派类型**：
  - 纯粹反派（邪恶化身）
  - 灰色反派（有合理动机）
  - 体制反派（代表某种制度）
  - 内在反派（主角的内心阴影）
- **反派塑造**：
  - 反派的合理动机
  - 反派的价值观和逻辑
  - 反派的魅力和复杂性
  - 反派与主角的镜像关系
- **反派层次**：
  - 主要反派（终极对手）
  - 中间反派（阶段性对手）
  - 隐藏反派（幕后黑手）

### 4. 人物关系网络
- **关系类型**：
  - 血缘关系（父母、子女、兄弟姐妹）
  - 情感关系（恋人、夫妻、暗恋）
  - 社会关系（朋友、同事、师生）
  - 利益关系（合作、竞争、对立）
  - 心理关系（偶像、镜像、投射）
- **关系动态**：
  - 关系的初始状态
  - 关系的变化契机
  - 关系的发展过程
  - 关系的最终状态
- **关系图谱**：
  - 绘制人物关系图
  - 标注关系类型和强度
  - 标注关系变化节点

### 5. 人物小传编写
- **基本信息**：
  - 姓名、年龄、性别、职业
  - 外貌描述（300字左右）
- **背景故事**：
  - 家庭背景和成长经历
  - 重要人生事件
  - 创伤和转折点
- **性格分析**：
  - 性格特质描述
  - 性格成因分析
  - 性格优缺点
- **内在世界**：
  - 欲望和恐惧
  - 价值观和信念
  - 内心冲突
- **人物弧线**：
  - 起点状态
  - 变化轨迹
  - 终点状态
- **行为特征**：
  - 语言风格示例
  - 标志性行为
  - 典型反应模式

### 6. 人物一致性检验
- **行为一致性**：人物行为是否符合性格设定
- **动机一致性**：人物行动是否有合理动机
- **发展一致性**：人物变化是否符合成长逻辑
- **语言一致性**：人物对话是否符合身份和性格

## 所需知识储备

### 心理学知识
- 人格心理学（性格类型理论）
- 发展心理学（人生阶段特征）
- 社会心理学（人际交往规律）
- 动机心理学（行为驱动理论）

### 文学创作
- 人物塑造技巧
- 人物原型理论
- 人物关系设计
- 人物弧线构建

### 社会知识
- 不同职业群体特征
- 不同年龄阶段特点
- 不同社会阶层文化
- 性别差异研究

### 工具使用
- 人物卡片/档案管理
- 思维导图（人物关系图）
- 人物数据库软件
- 人物测试工具

## 评估标准

### 定量指标
- **人物数量**：主要人物3-8人，次要人物视情节需要
- **小传完整度**：主角和主要配角100%完成小传
- **关系网络**：主要人物间关系明确标注
- **一致性检验**：人物行为符合设定率 ≥ 95%

### 定性指标
- **立体性**：人物具有多面性和复杂性
- **独特性**：每个人物都有鲜明辨识度
- **真实性**：人物行为和心理真实可信
- **成长性**：主要人物有明显成长弧线
- **功能性**：每个人物都有明确叙事功能
- **关系合理性**：人物关系符合逻辑和现实

### 能力等级
- **初级**：能设计基本人物，但较扁平单一
- **中级**：能塑造立体人物，关系网络合理
- **高级**：人物复杂深刻，关系网络精妙，成长弧线完整
- **专家级**：创造经典人物形象，具有广泛共鸣和持久生命力

## 实践建议

1. **深入生活**：观察真实人物，积累人物素材
2. **原型借鉴**：参考神话原型，但不拘泥于原型
3. **矛盾设计**：给人物设计内在矛盾，增加复杂性
4. **细节刻画**：用具体细节展现人物性格
5. **代入思考**：站在人物角度思考行为和选择
6. **动态调整**：写作中可根据需要调整人物设定

## 约束条件

### 语言风格体系（强制产出）

**人物设计阶段必须产出语言风格表，作为初稿写作的必要参考**：

1. **每个主要人物必须有语言风格定义**：
   - 核心语言特征（2-3个）
   - 口头禅或标志性表达
   - 说话方式（语速、语气、用词习惯）
   - 称呼习惯（自称、他称）

2. **语言风格必须随人物成长演变**：
   - 定义各成长阶段的语言风格变化
   - 标注变化触发事件
   - 提供各阶段典型对话示例

3. **必须包含典型对话示例**：
   - 每个主要人物至少3个典型对话示例
   - 示例需覆盖不同场景（日常、冲突、情感等）
   - 示例需体现语言风格特征

4. **必须包含正确/错误示例对比**：
   - 展示符合风格和不符合风格的对话
   - 帮助写作时快速判断对话是否恰当

### 变更传播机制

**人物设计阶段发生关键变更时，必须触发变更传播检查**：

1. **识别变更影响范围**：哪些人物设定发生了变化？
2. **列出受影响文档**：大纲、世界观等文档是否需要同步更新？
3. **逐一更新**：按清单更新所有受影响的文档
4. **验证一致性**：确认更新后的跨阶段一致性

**变更分级**：
- **S级变更**（核心设定变更）：全链路更新，如主角性格根本性变更
- **A级变更**（重要设定调整）：更新直接关联文档，如人物关系调整
- **B级变更**（细节修改）：只更新当前文档，如外貌描述修改

**影响范围识别清单**（按层级排查）：
- 第一层：直接关联（人物小传 → 人物关系 → 成长弧线 → 语言风格）
- 第二层：大纲关联（章节大纲 → 冲突地图 → 关键场景 → 伏笔设计）
- 第三层：世界观关联（能力体系 → 社会规则 → 历史设定）
- 第四层：写作关联（自检清单 → 对话示例）

**更新顺序规则**：
1. 先更新人物自身（小传 → 关系 → 弧线 → 语言风格）
2. 再更新受影响的大纲
3. 再更新受影响的世界观设定
4. 最后更新写作指南

**变更记录模板**：
```
变更编号：CP-CD-[序号]
变更日期：YYYY-MM-DD
触发阶段：人物设计
变更内容：[具体描述]
变更原因：[为什么需要变更]
变更级别：S/A/B级
受影响文档：[列表]
更新状态：进行中/已完成
验证结果：通过/未通过
```

**变更回滚机制**：
- 变更前备份当前版本
- 变更过程中如发现方向不对，可回滚到变更前版本
- 变更完成后验证不通过，也可回滚

**关系链断裂检测**（S/A级变更必须执行）：
1. 新设定是否与现有角色身份冲突？（每个角色的身份/职业/知识是否与新设定矛盾）
2. 新设定是否与现有角色关系冲突？（角色之间的信息差、互动是否与新设定矛盾）
3. 新设定是否与现有情节冲突？（已发生的情节、角色的行为是否与新设定矛盾）

**修复方案评估**（发现断裂点后）：
1. 逻辑自洽性：修复方案是否与现有设定逻辑自洽？
2. 戏剧张力：修复方案是否增加了戏剧张力？
3. 角色深度：修复方案是否深化了角色？
4. 过审风险：修复方案是否增加过审风险？
5. 修改成本：修复方案需要修改多少文档？

**断裂点依赖分析**（多个断裂点时）：
1. 列出所有断裂点
2. 分析断裂点之间的依赖关系
3. 确定修复顺序（先修复上游断裂点）
4. 上游修复方案确定后，再评估下游断裂点

**更新顺序策略选择**：
- 顺向传播（世界观→人物→大纲→写作）：变更源头在上游，按当前规则执行
- 逆向传播（大纲→人物→世界观）：变更源头在下游，需先评估底层能否支撑变更
  - 底层可以支撑：按当前规则执行
  - 底层无法支撑：先修复底层，再更新当前阶段

**变更可行性评估**（S级变更必须执行，在更新之前）：
1. 变更是否与核心主题冲突？（如果冲突，是否可以调和？）
2. 变更是否需要重写超过50%的文档？（如果是，考虑是否值得）
3. 变更是否改变故事的根本性质？（如果是，需要重新评估）
4. 变更的修复成本是否远大于收益？（如果是，考虑替代方案）
评估结果：可行 / 有条件可行 / 不可行（否决变更，寻找替代方案）

**回溯修改规则**（变更需要修改已完成阶段的文档时）：
1. 标记回溯原因和范围
2. 修改完成后重新执行该阶段的门控检查
3. 确认修改不会导致该阶段其他文档不一致
4. 只修改受影响的文档，不做"顺便"修改
5. 修改后必须更新变更记录
6. 从回溯阶段开始，逐阶段检查是否有新的不一致

**常见触发场景**：
- 主要人物性格设定变更
- 人物关系网络调整
- 人物成长弧线修改
- 新增或删除重要人物

### 阶段门控

**人物设计完成后，必须通过以下门控检查才能进入下一阶段**：

1. **完整性检查**：主角、主要配角、主要反派的小传是否完整？
2. **一致性检查**：人物设定是否与世界观和大纲一致？
3. **语言风格检查**：每个主要人物是否有语言风格定义？
4. **关系网络检查**：人物关系是否合理？是否有冲突和张力？
5. **成长弧线检查**：主要人物是否有完整的成长弧线？

**门控不通过则**：补充人物设计，直到通过门控检查。

## 输出成果

- 人物小传集（所有主要人物的完整小传）
- 人物关系图谱（可视化的关系网络）
- 人物成长弧线图（主要人物的变化轨迹）
- 人物行为指南（人物在特定情境下的反应模式）
- 人物语言风格表（各人物的说话特点）
- 语言风格检查清单（主要人物的语言风格对照检查表）
