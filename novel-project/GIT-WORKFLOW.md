# Git版本控制工作流程

## 📋 版本控制策略

### 提交规范

本项目采用规范的Git提交信息格式：

```
<type>(<scope>): <subject>

<body>

<footer>
```

#### Type类型

| Type | 说明 | 示例 |
|------|------|------|
| feat | 新功能 | feat(chapter): 添加第6章内容 |
| fix | 修复问题 | fix(plot): 修复情节逻辑漏洞 |
| docs | 文档变更 | docs(readme): 更新项目说明 |
| style | 格式调整 | style(chapter): 调整章节格式 |
| refactor | 重构 | refactor(structure): 重构五卷式框架 |
| test | 测试 | test(skill): 测试skill链功能 |
| chore | 构建/工具 | chore(git): 更新.gitignore |

## 🔄 分支策略

### 主分支

| 分支 | 说明 | 保护 |
|------|------|------|
| main/master | 主分支，稳定版本 | 是 |
| develop | 开发分支 | 是 |

### 功能分支

| 分支命名 | 说明 | 示例 |
|---------|------|------|
| feature/volume-N | 卷次创作 | feature/volume-1 |
| feature/chapter-N | 单章创作 | feature/chapter-6 |
| fix/xxx | 问题修复 | fix/plot-logic |
| docs/xxx | 文档更新 | docs/readme |

## 📝 提交模板

### 章节创作提交

```
feat(chapter): 完成第N章 - [章节标题]

- 核心事件：[事件描述]
- 字数：[X]字
- 幽默点：[数量]个
- 伏笔：[伏笔ID]

Closes #issue-number
```

### 案件完成提交

```
feat(case): 完成案件N - [案件名称]

- 案件类型：[类型]
- 涉及章节：第X-Y章
- 核心谜题：[谜题描述]
- 真相：[真相描述]
- 推理设计：[推理模式]

Closes #issue-number
```

### 卷次完成提交

```
feat(volume): 完成第N卷 - [卷名]

- 涉及章节：第X-Y章
- 案件数量：[N]个
- 核心主题：[主题]
- 人物成长：[成长描述]
- 高潮章节：第N章

Closes #issue-number
```
