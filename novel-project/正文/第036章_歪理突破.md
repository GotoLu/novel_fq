# 第三十六章 歪理突破

"歪理找方向，精密做验证。"

李大嘴在歪理笔记的扉页上写下这行字，然后在下面加了一个括号：

**（赵精英说的。不对，是我说的。他肯定不会用"歪理"这个词。）**

他合上笔记本，看着对面的赵精英。

赵精英正在打电话。电话那头是市局的技术处，他要求调一台专业视频分析设备过来。

"对，帧级分析软件，"赵精英说，"能做帧间差异对比和时间戳校验的那种。对，今天就要。"

他挂了电话，看着李大嘴。

"你说监控在说谎，我需要证明它怎么说的谎。逐帧看不行，因为如果画面本身就是拼接好的，逐帧看不出来。就像你把两张照片拼在一起，只要接缝处理得好，肉眼看不出来。"

"那什么能看出来？"李大嘴问。

"数据，"赵精英说，"肉眼看的是画面，数据看的是底层。如果画面被拼接过，帧与帧之间的数据特征会有细微差异。人眼看不到，但软件可以。"

他打开笔记本电脑，调出监控录像的文件夹。

"在等专业设备来之前，我先做一件事。"

他右键点击一个视频文件，选择"属性"，然后点开"详细信息"选项卡。

"这是7月8日展厅1号摄像头的录像文件。你看，文件创建时间7月9日7:15，文件大小2.3GB，视频编码H.264，分辨率1920×1080，帧率25fps，时长12小时31分钟。"

他一个文件一个文件地点开，记录每个摄像头的文件属性。

李大嘴站在旁边看，不太懂，但觉得赵精英做事情确实精密。每一个数据都记录在笔记本上，整整齐齐。

突然，赵精英的手停住了。

"这个不对，"他说。

---

"哪个不对？"李大嘴凑过去。

赵精英指着屏幕上的两个文件属性窗口。左边是展厅1号摄像头的录像，右边是走廊3号摄像头的录像。

"你看文件大小，"赵精英说，"1号摄像头，12小时31分钟，2.3GB。3号摄像头，12小时31分钟，2.28GB。"

"差了0.02GB，"李大嘴说，"这有什么问题？"

"单独看没问题，"赵精英说，"但你看这个。"

他点开了展厅2号摄像头的文件属性。

"2号摄像头，12小时31分钟，2.31GB。"

他又点开了展厅3号摄像头。

"3号摄像头，12小时31分钟，2.29GB。"

他快速点开了所有14个摄像头的文件属性，把文件大小列在笔记本上：

| 摄像头 | 时长 | 文件大小 |
|--------|------|---------|
| 展厅1号 | 12:31 | 2.30GB |
| 展厅2号 | 12:31 | 2.31GB |
| 展厅3号 | 12:31 | 2.29GB |
| 展厅4号 | 12:31 | 2.30GB |
| 展厅5号 | 12:31 | 2.28GB |
| 展厅6号 | 12:31 | 2.30GB |
| 展厅7号 | 12:31 | 2.31GB |
| 展厅8号 | 12:31 | 2.30GB |
| 走廊1号 | 12:31 | 2.28GB |
| 走廊2号 | 12:31 | 2.29GB |
| 走廊3号 | 12:31 | 2.28GB |
| 走廊4号 | 12:31 | 2.29GB |
| 大门1号 | 12:31 | 2.30GB |
| 大门2号 | 12:31 | 2.29GB |

"14个摄像头，同样的分辨率，同样的帧率，同样的时长，"赵精英说，"文件大小应该在2.28到2.31之间波动，这是正常的压缩差异。"

"那问题在哪？"

赵精英指着展厅2号和展厅7号。

"这两个，2.31GB。比其他的大了0.01到0.03GB。"

"0.03GB？"李大嘴算了一下，"大概30MB？"

"30MB，"赵精英说，"在2.3GB的文件里，30MB只占1.3%。正常情况下，这完全可以忽略。但如果画面被拼接过，拼接点会产生额外的数据。因为两段不同时间录制的画面，帧间的压缩特征不同，编码器会在拼接点附近增加码率来保证画质。"

他看着那两个数字。

"2号摄像头覆盖展厅正门方向，7号摄像头覆盖展厅侧门方向。这两个位置的文件比其他大了30MB左右。"

"你的意思是，这两个摄像头的录像被拼接过？"

"我不确定，"赵精英说，"文件大小的差异在正常范围内，不能单独作为证据。但如果帧间差异分析也发现异常，那就可以确认了。"

---

下午，市局技术处的人到了。

带来了一台便携式视频分析工作站，装了专业软件。赵精英把14个摄像头的录像全部导入，开始做帧间差异分析。

软件的工作原理赵精英解释给李大嘴听："正常录制的视频，相邻两帧之间的像素差异很小，因为时间只过了1/25秒，画面变化不大。软件会计算每一帧和前一帧的像素差异值，生成一条曲线。如果画面是连续录制的，曲线应该是平滑的。如果画面被拼接过，拼接点处的像素差异会突然增大，曲线上会出现尖峰。"

"就像两块布缝在一起，"李大嘴说，"缝线的地方总会比其他地方厚一点。"

赵精英看了他一眼："差不多是这个意思。"

他点击"开始分析"，屏幕上出现了14条曲线。

大部分曲线都很平滑，像心电图上平稳的心跳。

但展厅2号和展厅7号的曲线上，各出现了一个尖峰。

2号摄像头的尖峰出现在3:47:12。

7号摄像头的尖峰出现在3:47:15。

两个尖峰相隔3秒。

"3:47，"赵精英说，"和瓶子消失的时间吻合。3:47之前，画面是连续录制的。3:47之后，画面也是连续录制的。但3:47附近，画面被拼接过。"

他放大了2号摄像头的尖峰区域，逐帧查看。

3:47:11，瓶子在展柜里。画面正常，像素差异值0.3%。

3:47:12，瓶子还在展柜里。但像素差异值跳到了2.7%。

3:47:13，瓶子还在。像素差异值回到0.4%。

"3:47:12这一帧，"赵精英说，"和前一帧的像素差异是正常值的9倍。肉眼看不出来，因为画面内容几乎一样，瓶子还在展柜里。但数据层面，这一帧和前一帧的压缩特征不同。"

"为什么不同？"李大嘴问。

"因为这两帧不是同一次录制的，"赵精英说，"3:47:12之前是7月8日凌晨的实时录像。3:47:12之后是另一段录像，被拼接到了这里。"

他继续分析7号摄像头，结果一样：3:47:15出现尖峰，拼接点。

"两个摄像头，同一时间段，都出现了拼接痕迹，"赵精英说，"这不是巧合。"

---

赵精英继续深挖。

他用软件提取了拼接点前后各10分钟的画面，做时间戳校验。

"视频文件的时间戳有两种，"他解释，"一种是显示在画面上的时间水印，人眼可以看到。另一种是嵌入在视频数据里的元数据时间戳，记录每一帧的精确录制时间。正常情况下，这两种时间戳应该完全一致。"

他调出2号摄像头3:37到3:57的元数据时间戳。

3:37:00，元数据时间戳7月8日03:37:00。一致。

3:38:00，元数据时间戳7月8日03:38:00。一致。

3:39:00，元数据时间戳7月8日03:39:00。一致。

一直到了3:47:12，拼接点。

3:47:12，画面水印显示7月8日03:47:12。元数据时间戳显示7月8日03:47:12。

一致。

赵精英皱了皱眉。

"元数据时间戳也被修改了，"他说，"拼接的人不仅拼接了画面，还修改了元数据时间戳，让两种时间戳保持一致。"

"那怎么证明拼接过？"李大嘴问。

赵精英想了想，换了一个分析方式。

"如果元数据时间戳被修改过，修改操作本身会留下痕迹。就像伪造文件，内容可以伪造，但纸张的年份伪造不了。我检查一下文件的修改历史。"

他调出文件的系统元数据，查看创建时间、修改时间和访问时间。

"创建时间7月9日7:15，这是保洁人员发现失窃后保安调取录像的时间。修改时间……"

他停住了。

"7月8日03:52:17。"

李大嘴算了一下："3:52？但瓶子3:48就消失了。3:52修改文件，是在瓶子消失4分钟之后？"

"对，"赵精英说，"而且这个修改时间只出现在2号和7号摄像头的文件上。其他12个摄像头的文件，修改时间和创建时间一致，都是7月9日7:15。"

他靠在椅背上，看着屏幕上的数据。

"2号和7号摄像头，文件大小异常，帧间差异异常，修改时间异常。三个异常同时出现在这两个摄像头上。"

"这意味着什么？"

"意味着作案者在3:52左右，远程访问了硬盘录像机，修改了2号和7号摄像头的录像文件，"赵精英说，"他把3:47之后的真实画面替换成了提前录制好的'空展厅'画面。所以3:47之前，监控拍到的是真实的凌晨画面，瓶子还在。3:47之后，监控拍到的是提前录制的空展厅画面，瓶子已经不在了。"

"提前录制的？"李大嘴问，"什么时候录的？"

赵精英调出了帧间差异的完整曲线。

"2号摄像头3:47:12之后的画面，帧间差异曲线非常平稳，比3:47之前还要平稳。这说明3:47之后的画面是在更安静的环境下录制的，可能是白天闭馆前的某个时段。"

他看了看画面上的光线特征。

"3:47之后的画面，展厅灯光是凌晨的昏暗模式。但帧间差异的平稳程度更接近白天的录制。我推测，作案者在白天录制了一段'空展厅+凌晨灯光'的画面，然后在3:47之后替换了真实录像。"

"等一下，"李大嘴说，"你说3:47之后替换了。那3:47之前呢？2:17到3:47，一个半小时的录像，有没有问题？"

赵精英检查了2:17到3:47的帧间差异曲线。

"这段曲线整体平稳，没有拼接尖峰。但……"

他放大了曲线，仔细看。

"2:17到3:00，帧间差异平均值0.31%。3:00到3:47，帧间差异平均值0.28%。"

"3:00之后更低了？"李大嘴想起王刚的话——凌晨3点听到走廊有声音。

"对，"赵精英说，"3:00之后帧间差异降低了0.03%。这个差异很小，在正常范围内。但如果3:00之后画面被替换成了提前录制的版本，那帧间差异降低就说得通了——提前录制的画面更稳定，因为录制时展厅里确实没有人。"

"也就是说，"李大嘴慢慢说，"3:00之后，2号和7号摄像头的画面可能就已经被替换了？"

"有可能，"赵精英说，"但0.03%的差异太小，不能作为确凿证据。只能说是疑点。"

他看着屏幕上的数据，沉默了几秒。

"但3:47的拼接点是确凿的。2号和7号摄像头，同一时间，同一手法。这不是系统故障，这是人为操作。"

---

李大嘴翻开歪理笔记，在推理链下面继续写：

**"监控说谎→谁能让监控说谎→最了解监控系统的人→安装和调试监控系统的人→鸿运安防的技术人员"**

**"监控怎么说谎→替换录像画面→提前录制空展厅→3:47拼接→3:52修改文件→远程访问硬盘录像机"**

**"谁有这个能力→能远程访问硬盘录像机的人→知道系统密码和架构的人→安装和调试系统的人→鸿运安防的技术人员"**

他看着这两条推理链，一条指向"谁"，一条指向"怎么"。

歪理推理找到了"谁"的方向。

精密推理验证了"怎么"的方式。

两条链在同一个终点汇合：鸿运安防的技术人员。

他合上笔记本，看着赵精英。

赵精英正在打电话，声音很低，但李大嘴听到了几个关键词："孙磊""马文才""调查""监控"。

赵精英挂了电话，转过身来。

"我让王小美去查孙磊和马文才的详细背景了，"他说，"包括他们的手机定位记录、银行流水、社交关系。如果他们中有人是作案者，一定会有其他痕迹。"

他顿了顿。

"还有一件事。硬盘录像机是独立网络，不连接外网。如果要远程访问，必须在内网操作。但文化中心的内网只有保安室和办公区的电脑可以访问。"

"也就是说，"李大嘴说，"作案者要么在现场，要么有办法从外网进入内网。"

"对，"赵精英说，"但如果作案者是鸿运安防的技术人员，他在安装调试期间，完全有机会设置一个后门，让自己可以从外网远程访问硬盘录像机。"

"后门？"李大嘴不太懂。

"就是系统里隐藏的远程访问通道，"赵精英说，"安装系统的人可以留一个只有自己知道的后门。系统正常运行时，后门不会被发现。但需要的时候，他可以通过后门远程操控整个监控系统。"

李大嘴想了一下，说："就像装修房子的时候，装修工人偷偷配了一把钥匙。"

赵精英看了他一眼，点了点头。

"对。就像配了一把钥匙。"

---

李大嘴站在文化中心门口，看着那块"安保系统由鸿运安防提供"的小牌子。

他已经看了这块牌子三天了。

第一天，他多看了一眼，没说什么。

第二天，他又多看了一眼，还是没说什么。

第三天，他终于知道这块牌子在告诉他什么了。

不是"安保系统由鸿运安防提供"。

是"安保系统由鸿运安防控制"。

提供和控制，一字之差。

提供是服务，控制是权力。

谁控制了监控系统，谁就控制了监控看到的东西。

谁控制了监控看到的东西，谁就能让监控说谎。

谁能让监控说谎，谁就能在监控下消失。

他翻开歪理笔记，在最后一页写下完整的推理链：

**"不可能→可能→怎么可能→监控说没人→监控会不会骗人→谁能让监控骗人→装监控的人→鸿运安防→监控说谎→替换画面→拼接+后门→真相"**

他看着这行字，长长地呼了一口气。

歪理推理，从"不可能"走到了"真相"。

中间经过的每一步，都是歪的。

但歪着歪着，就走到了正道上。

也许这就是歪理推理的本质：不走直路，但能到终点。

他合上笔记本，转身走进文化中心。

赵精英还在里面分析数据，王小美还在外面调查背景。

他呢？他的活干完了。

歪理找方向，精密做验证。

方向找到了，验证做完了。

接下来，该收网了。
